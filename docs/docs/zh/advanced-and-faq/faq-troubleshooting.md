# 常见问题与排障指南 (FAQ & Troubleshooting)

## 模块导读
在使用基于本地文本解析的复杂系统中，偶尔会遇到由于格式不规范或同步延迟导致的局部异常。本页汇总了用户最常遇到的几类现象及对应的自我排查清单。

如果您遇到“卡片出不来”、“数据不更新”等疑惑，请尝试按照以下指南进行快速定位。

## 现象 1：我写了卡片/挖空，但在牌组树里看不到
这通常是由于 Markdown 解析规则未能匹配您的输入导致的。
**排查清单：**
1. **检查分隔符设置**：前往插件设置 -> `Flashcards` 标签，确认您输入的分隔符（如 `::`）与设置中定义的规则完全一致（注意中英文冒号的区别）。
2. **检查来源开关**：如果您使用了高亮 `==文本==` 作为挖空，请确保设置中已明确开启“将高亮作为 Cloze 来源”的开关。
3. **隔离排查法**：不要在几千字、排版极其复杂的笔记里找 Bug。请新建一个空白笔记，只写一行最简单的标准问答测试（如 `A::B`），看能否正常解析。如果可以，说明是您原笔记中的某处复杂排版（如缺少空行、代码块嵌套错误）阻断了解析器。

## 现象 2：队列数字异常，或复习时报错“找不到笔记”
这通常发生在您大规模移动了文件、重命名了文件夹，导致系统缓存的数据路径失效。
**排查清单：**
1. **快速等待（冷笑话）**：如果是少量修改，等待几秒钟，自动增量同步会修复它。
2. **触发全量重建**：打开命令面板，执行 `Syro: Rebuild Cache (Full Parse)`。这通常能解决 95% 的“状态与文本不一致”问题。
3. **清理幽灵数据**：如果某篇笔记已被物理删除，但复习界面依然固执地弹出它的卡片，可执行命令 `Syro: Clean Ghost Cards` 强制清理无效引用。

## 现象 3：感觉底层的复习算法不起作用
**排查清单：**
1. **新旧算法交替**：请确认您在设置中启用的是 FSRS 算法。部分带有旧版兼容标签的参数滑块，若未经激活，调整它们并不会产生实际效果。
2. **上限拦截**：如果您感觉某些该复习的卡片一直不出现，请前往牌组树点击对应牌组的齿轮（⚙️）图标，检查“每日复习上限（Reviews/Day）”是否设置得过低，导致卡片被系统拦截在了今天的大门外。

## 进阶帮助与反馈
如果您穷尽了以上步骤依然无法解决问题：
- 您可以在插件设置的 `UI` 标签中开启 **调试模式（Verbose Logging）**，此时按 `Ctrl/Cmd + Shift + I` 打开开发者工具，控制台会输出详细的解析错误日志，帮助您锁定具体是哪一行文本引发了崩溃。
- 欢迎携带有问题的且确保它将不会暴露您的隐私的“最小 Markdown 文本片段”以及您的错误日志，前往社区或代码仓库提交 Issue 寻求支持。
