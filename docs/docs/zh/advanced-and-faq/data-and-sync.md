# 数据、同步与备份机制

## 模块导读
作为一款深度融合于 Obsidian 本地化理念的插件，Syro 绝不会将您的学习数据上传至任何闭源云端。本页旨在透明地向您解释调度数据存储在哪里、系统如何捕捉您的文本修改，以及如何安全地进行备份与迁移。

如果您在进行跨设备同步，或者对系统“如何记住复习进度”感到好奇，请参考本页内容。

## 1. 同步机制：系统如何懂你？
Obsidian 具有极高的文件操作自由度。为了确保复习卡片与原始文本不脱节，Syro 提供了一套稳健的同步方案：
- **自动增量同步（Auto-incremental Sync）**：这是默认且推荐的模式。当您修改一张闪卡的内容、添加一个新的填空或更改标签时，系统会在后台静默计算这些微小的差异，并实时更新复习数据库。您几乎感觉不到它的存在。
- **手动全量重建（Rebuild Cache）**：当您使用外部脚本批量修改了数百个文件，或使用第三方同步盘（如 iCloud、Syncthing）在另一台电脑上拉取了大量新文件后，自动同步可能会遗漏。此时，通过命令面板执行 `Syro: Rebuild Cache`，可强制系统全量扫描并对齐最新状态。

## 2. 存储模型：您的数据在哪儿？
Syro 的所有调度状态数据，均以纯文本 JSON 格式安全地存储在您的本地插件目录中（具体路径为：`.obsidian/plugins/syro/` 下）。
核心文件包括：
- `tracked_files.json`：承载所有闪卡的解析结果、FSRS 算法调度状态以及笔记追踪状态的主数据文件。
- `review_notes.json` / `review_commits.json`：独立存储笔记复习队列的 Timeline 历史进度与阅读上下文。
- *(您可能会看到带有 `.overlay` 后缀的临时文件，这是系统为了防止数据写入冲突和优化大文件写入速度而生成的增量缓存层，属于正常现象。)*

## 3. 备份与多端迁移建议
由于所有数据都严格存储在您的 Obsidian Vault（库）内：
- **常规备份**：只要您正常备份了整个 `.obsidian` 隐藏文件夹，Syro 的所有学习进度、卡片评分和算法状态都会被完整保留。
- **跨设备多端使用**：如果您使用官方的 Obsidian Sync 或第三方同步工具（如 Git、OneDrive），请务必确保同步规则**不要忽略** `.obsidian/plugins/syro/` 目录下的 `.json` 数据文件。否则您的复习进度将无法在手机和电脑间漫游。

*提示：在进行任何手动修改底层 JSON 数据文件的高危操作前，请务必先复制一份该文件夹作为安全备份。*
